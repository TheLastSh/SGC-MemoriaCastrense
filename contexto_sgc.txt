[STACK TECNOLÓGICO Y CONTEXTO GENERAL - SGC MEMORIA CASTRENSE]

1. RESUMEN DEL PROYECTO
El "SGC Memoria Castrense" es un Sistema Gestor de Contenidos (CMS) diseñado específicamente para digitalizar, almacenar, consultar y debatir sobre documentos históricos militares (como actas oficiales, mapas estratégicos, fotografías de época y registros de condecoraciones).

2. ARQUITECTURA DE SOFTWARE
- Paradigma: Programación Orientada a Objetos (POO).
- Arquitectura de Despliegue: Aplicación Monolítica (Frontend y Backend conviven en el mismo servidor para maximizar el SEO y minimizar la latencia).
- Patrones de Diseño: MVC (Modelo-Vista-Controlador) enriquecido con el Patrón de Servicios (Service Pattern) para aislar la lógica de negocio pesada de los controladores.

3. STACK TECNOLÓGICO (BACKEND)
- Lenguaje Core: PHP 8.2
- Framework Principal: Laravel 10
- Autenticación: Laravel Breeze / Session Guards (Bcrypt para encriptación de contraseñas).
- Lógica Transaccional: Uso extensivo de transacciones ACID (DB::transaction) y manejo de excepciones (try-catch) para asegurar la integridad de la base de datos al subir archivos pesados.

4. STACK TECNOLÓGICO (FRONTEND)
- Motor de Plantillas: Laravel Blade (Server-Side Rendering).
- Estilos (CSS): Tailwind CSS (Generación estática optimizada vía Vite).
- Interactividad (JS): Vanilla JavaScript.
- Componentes Visuales: Interfaz responsiva con temática institucional (tonos navy y dorados) usando iconos SVG (Heroicons).

5. BASES DE DATOS Y PERSISTENCIA (SQL)
- Entorno de Desarrollo (Local): SQLite (Base de datos embebida en un archivo para rápida iteración).
- Entorno de Producción (Cloud): PostgreSQL.
- Abstracción de Datos: Eloquent ORM (Manejo de relaciones 1:N y N:M usando claves primarias BIGINT y UUID según la entidad).
- Borrado Lógico: Implementación de Soft Deletes en los registros principales (los documentos se "archivan" pero nunca se borran físicamente de la base de datos).

6. ALMACENAMIENTO DE ARCHIVOS (FILE STORAGE)
Debido a que el sistema se despliega en plataformas PaaS con sistemas de archivos efímeros, la gestión de documentos físicos (PDFs, Imágenes JPG/PNG) está separada del servidor principal:
- Proveedor de Storage: Amazon S3 / Cloudinary (Almacenamiento de Objetos).
- Integración: Laravel Flysystem. La base de datos SQL no guarda el archivo en binario ni en una ruta local, sino que almacena la URL absoluta externa (`url_recurso` tipo TEXT) que apunta al bucket en la nube.

7. INFRAESTRUCTURA DE DESPLIEGUE
- Plataforma: Render (PaaS) u otro entorno en la nube compatible con servidores web modernos. El proyecto se empaqueta y se sirve de forma centralizada sin requerir la división física de los servidores frontend y backend.
